enum CaptureProcessing

Image processing.

Enumerator:
DP_IMG_PROC_DEFAULT 

Recommended processing. This is to acquire image as fast as possible. The image is suitable for feature extraction.

DP_IMG_PROC_PIV 

To request image which will be PIV compliant. Not every reader supports this mode. The image is suitable for feature extraction.

DP_IMG_PROC_ENHANCED 

To request visually enhanced image. The image is suitable for feature extraction.

DP_IMG_PROC_UNPROCESSED 

To request image which is not processed in any way. The image is NOT suitable for feature extraction.

enum ResultCode

Result codes:

DP_SUCCESS - Success

DP_NOT_IMPLEMENTED - Signals that particular function is not implemented.

DP_FAILURE - “Catch-all” generic failure code. Can be returned by all API calls in case of failure, when the reason of failure unknown or cannot be specified.

DP_NO_DATA - No data is available.

DP_MORE_DATA - Memory allocated by application is not enough to contain data which is expected.

DP_INVALID_PARAMETER - Returned when one or more parameters passed to the API call are not valid.

DP_INVALID_DEVICE - Returned when the current Reader object is not associated with a valid device.

DP_DEVICE_BUSY - Returned when API call cannot be completed because another operation is in progress.

DP_DEVICE_FAILURE - Returned in case of reader not working properly.

DP_INVALID_Fid - Returned when the image data received by the SDK is invalid.

DP_TOO_SMALL_AREA - Returned if the image is too small.

DP_INVALID_Fmd - Returned when the minutiae data received by the SDK is invalid.

DP_ENROLLMENT_IN_PROGRESS - Enrollment operation is in progress.

DP_ENROLLMENT_NOT_STARTED - Enrollment operation has not begun.

DP_ENROLLMENT_NOT_READY - Not enough in the pool of Fmds to create enrollment Fmd.

DP_VERSION_INCOMPATIBILITY - Returned when version of the deserialized data is greater than the installed wrapper’s version.
